# 週次食品動向報告書 運用手順書

## 概要

このツールは、毎週の経営会議向け食品動向報告書を自動生成します。
玉子価格と期間をExcelに入力するだけで、Web検索で最新情報を収集し、
Word文書として完成形を出力します。

### 4. 出力された報告書を確認

1. エクスプローラーで `work` フォルダを開く
2. **最新の `週次報告書_yyyymmdd_HHMMSS.docx`** をダブルクリックでWordで開く
3. 以下のチェック項目を確認:

#### チェックポイント

- [ ] タイトルが「食品動向 週次報告書」になっている
- [ ] 対象期間が正しく入力されている
- [ ] 玉子価格3都市が正しい（東京・大阪・名古屋）
- [ ] 野菜価格が13品目すべて揃っている
  - キャベツ、はくさい、レタス、ほうれんそう、ねぎ、きゅうり、トマト、なす、ピーマン、だいこん、にんじん、じゃがいも、たまねぎ
- [ ] 食品値上げ報道が記載されている（または「対象期間内の報道なし」）
- [ ] 厨房消耗品の値上げ報道が記載されている（同上）
- [ ] 食中毒情報が記載されている（同上）
- [ ] 総括が200字程度で書かれている

#### 内容に違和感がある場合の対処
- 期間外の情報が混じっている → 再実行（Web検索結果のばらつき）
- 野菜が13品目揃っていない → 再実行
- メタコメントが残っている → 再実行

## トラブルシューティング

### エラー: `PermissionError: [Errno 13] Permission denied: 'food_prices.xlsx'`

→ Excelを開いたまま実行している。Excelを完全に閉じてから再実行。

### エラー: `AuthenticationError: invalid x-api-key`

→ `.env` ファイルのAPIキーに問題あり。Anthropicコンソールで確認。

### エラー: 出力が途中で切れる

→ `weekly_report.py` の `max_tokens=8000` を増やす（例: 12000）。

### Web検索結果が古い・少ない

→ 同じコマンドで再実行（毎回違う結果になる）。

### 実行コスト

- **1回の実行で約$0.20〜0.40程度**（Web検索15回 + 8000トークン出力）
- 月4回実行で月額$1〜2程度
- 心配な場合はAnthropicコンソールで使用量確認

## 緊急時の対応

### Pythonが動かない場合

1. Anthropicコンソール（https://console.anthropic.com）でAPIが生きているか確認
2. それでも動かない場合は従来のClaudeチャット手動実行に切り替え

### 過去の報告書を見たい

1. VSCodeのソース管理パネルでコミット履歴を確認
2. または `週次報告書_yyyymmdd_HHMMSS.docx` ファイルを過去の日付から探す
